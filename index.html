Détails techniques (résumé)

Chaîne canonique : v1|{type}|{date}|{juridiction}|{canal}|{compteur_local}|{secret_salt}

Hachage : HMAC‑SHA256(secret, canon) → hex → 7 premiers → + lettre type.

Idempotence : si le même v1|type|date|jur|canal a déjà un ID, on renvoie l'ID existant.

Unicité locale : compteur local par couple (type, date, juridiction, canal). Collisions rares, mais gérées par incrément.

Limites : en multi‑poste sans backend partagé, l'unicité n'est garantie que par instance.
